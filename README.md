# JAVA-ll
การอัปเดตเพิ่มเติมท้ายทดลอง
Custom Exception - QuestException: สร้างคลาสข้อยกเว้นขึ้นมาเองเพื่อจัดการกรณีที่ผู้ใช้ระบุ ID เควสต์ที่ไม่มีอยู่จริง 
แทนการคืนค่า null แบบเดิมทำให้ระบบมีความปลอดภัยและแจ้งเตือนผู้ใช้ได้อย่างชัดเจนผ่านบล็อก try-catch
Interface implementation - Rewardable: มีการสร้าง Interface ชื่อ Rewardable เพื่อกำหนดการให้รางวัล 
โดยบังคับให้ทั้ง DailyQuest และ StoryQuest ต้องมีวิธีการคำนวณคะแนนตามกฎของตัวเอง  ช่วยให้โค้ดรองรับการขยายประเภทเควสต์ใหม่ๆ
วิธีการทดสอบระบบ
การทดสอบกรณีสำเร็จ: เลือกเมนู 2 แล้วใส่ ID ของ DailyQuest เช่น D-29-1 ระบบจะเช็คค่าพลังและให้รางวัลคะแนนทันที 
การทดสอบกรณีล้มเหลว: ลองเลือก StoryQuest ที่ต้องการค่าพลังสูง เช่น logic > 60 ในขณะที่ตัวละครมีค่าพลังไม่ถึง ระบบจะบันทึกสถิติในหมวด Failed 
การทดสอบ Error: ลองกรอกเมนูที่ไม่ใช่ตัวเลข หรือใส่ ID เควสต์ที่ไม่มีอยู่จริง ระบบจะแสดงข้อความ Error จาก Exception ที่สร้างขึ้นโดยโปรแกรมไม่ล่ม
